## Краткое объяснение реализации

### Первое это генерация точек
если данные генерируются независимо друг от друга, то график будет похож на штрих код.
а вот если генерируемые числа накапливать и добавлять в лист, то картинка будет интереснее.

### Второе это само сглаживание Гаусса
для функции, которая определяет вес точки (её влияние на график), я взял одномерный вариант поскольку у меня линия

сама функция сглаживания помимо листа чисел принимает 2 параметра:
 * windowSize - отвечает за то, какой диапазон точек вокруг выбранной мы должны усреднить (соответственно чем больше этот параметр тем больше сглаживания и меньше деталей)
 * sigma - среднеквадратичное отклонение, нужно для определения веса, тоже чем он больше тем больше сглаживание

в функции 2 цинка:
* первый - проходит по каждому числу
* второй - проходит по соседям числа и усредняет смотря на них за счёт нахождения среднего веса промежутка
  
### Третье это отрисовка
тут всё просто:
1. соединяю кисточкой соседние точки по их координатам
2. отрисовываю каждую пятидесятую точку для красоты
3. всё масштабируется и делаются отступы
4. рисуются лэйблы с максимальным и минимальным значением получившейся функции
